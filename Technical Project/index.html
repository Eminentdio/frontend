Introduction

What is Frontend Development?

You can categorize website development into two areas: front end and back end. The front end is what you see on the screen — menus, graphics, and other features. You’ll also see the front end called client-side because that’s who is interacting with it. Front-end developers are primarily responsible for the site’s appearance.

What you should already know

A front-end developer uses code that affects how a website looks and how a user interacts with its webpages. That includes simple elements such as buttons that take a visitor to different places within the website, as well as input methods, like a contact form. The programming languages that a front-end developer uses can create a static webpage or a dynamic one. Programming languages they use include:

  • HTML
  • CSS
  • Javascript

In addition, you should have known how internet works. You have understood the relationship between DNS, IP and IP Address etc.

Hypertext Markup Language - HTML

HTML stands for Hyper Text Markup Language. It is the standard markup language used to create web pages. HTML consists of a series of elements that tell the browser how to display the content. Other technologies such as CSS and JavaScript are generally used to describe a web page's appearance/presentation or functionality/behavior, respectively.

HTML is the skeleton of every website. It is the basic structure of every website you are seeing out there. Examples of hmtl codes are < head> < h1> < p> < table> < form action="">

Cascading Style Sheet - CSS

Cascading Style Sheets (CSS) is a style sheet language used for specifying the presentation and styling of a document written in a markup language such as HTML or XML (including XML dialects such as SVG, MathML or XHTML). CSS is a cornerstone technology of the World Wide Web, alongside HTML and JavaScript.

CSS is designed to enable the separation of content and presentation, including layout, colors, and fonts. This separation can improve content accessibility; provide more flexibility and control in the specification of presentation characteristics; enable multiple web pages to share formatting by specifying the relevant CSS in a separate .css file, which reduces complexity and repetition in the structural content; and enable the .css file to be cached to improve the page load speed between the pages that share the file and its formatting.

CSS can be written in three different ways:

  • Inline: Which allows the inclusion of CSS rules inside an HTML element by introducing a style attribute. e.g < p style="color: red;">Hello< /p >, will render 'Hello' in colour red.
  • Internal: Which allows the inclusion of CSS rules in the < head>< /head> by introducing < style> < /style >.
  • External: Which allows the inclusion of a file which contains CSS rules inside the HTML file with the introduction of < link > in the head

You can learn more about CSS by reading up on W3 School.

Javascript

JavaScript is a high-level, often just-in-time compiled language that conforms to the ECMAScript standard.[11] It has dynamic typing, prototype-based object-orientation, and first-class functions. It is multi-paradigm, supporting event-driven, functional, and imperative programming styles. It has application programming interfaces (APIs) for working with text, dates, regular expressions, standard data structures, and the Document Object Model (DOM).

The use of JavaScript has expanded beyond its web browser roots. JavaScript engines are now embedded in a variety of other software systems, both for server-side website deployments and non-browser applications.

An example of a JS code is alert('Hello') which will prompt a message on the screen of the computer.

When Have I become an expert?

As a frontend developer, becoming an expert is a gradual process that requires consistent effort and dedication. There is no fixed timeline or criteria for becoming an expert. However, some common factors that can help you become an expert include:

  1. Experience: Gaining experience in the field of frontend development is crucial to becoming an expert. According to a forum post on FreeCodeCamp, a basic requirement for being a senior frontend developer is 5 years of employment as a frontend developer.
  2. Continuous Learning: Frontend development is a constantly evolving field, and staying up-to-date with the latest technologies and trends is essential to becoming an expert. You can achieve this by attending conferences, reading blogs, and taking online courses.
  3. Building a Portfolio: Building a portfolio of your work can help you showcase your skills and expertise to potential employers. You can include your best projects, code samples, and other relevant information in your portfolio
  4. Networking: Networking with other frontend developers can help you learn from their experiences and gain new insights into the field. You can join online communities, attend meetups, and participate in forums to connect with other developers.

Remember, becoming an expert is a journey, not a destination. Keep learning, stay curious, and don't be afraid to take on new challenges. Good luck.
